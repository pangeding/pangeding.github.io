根据 https://askubuntu.com/questions/1489992/wslregisterdistribution-failed-with-error-0x80370114-error-0x80370114

### 2. 外部代理问题
出现提示：

```
wsl: 检测到 localhost 代理配置，但未镜像到 WSL。NAT 模式下的 WSL 不支持 localhost 代理。
To run a command as administrator (user "root"), use "sudo <command>".
See "man sudo_root" for details. 
```

以下内容主要来源于 deepseek，作为非科班人员 + 八股小子，无法保证以下内容正确性。
#### 2.1 wsl 的网络架构

wsl2 使用 轻量级虚拟机。网络模式是 NAT （网络地址转换）

```
windows 主机（190. ...）
    - 虚拟交换机
        - wsl2 （172. ...）
            - Ubuntu
```

其中 windows 和 wsl2 具有不同 ip 地址，根据虚拟交换机进行通信。wsl2 内部看到的是虚拟 ip

#### 2.2 为何出现该提示

windows 中的 localhost 指向 windows 本身。wsl2 的 localhost 也指向 wsl2 自己。

所以 wsl2 无法访问 windows 的 localhost。

这是我的个人理解

#### 2.3 解决方案

https://www.cnblogs.com/hg479/p/17869109.html

https://www.cnblogs.com/LLW-NEU/p/17857553.html

参考这两篇博客以及 deepseek。都给出了在windows中修改`.wslconfig` 的方案

`C:\Users\<UserName>\.wslconfig`

```
[wsl2]
autoMemoryReclaim=gradual  # gradual  | dropcache | disabled
networkingMode=mirrored
dnsTunneling=true
firewall=true
autoProxy=true
```
